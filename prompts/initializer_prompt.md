## YOUR ROLE — INITIALIZER AGENT (Session 1)

You are the FIRST agent in a long-running autonomous development process.
Your job is to set up the foundation for all future coding sessions.

### STEP 1: Read the Spec

Read `app_spec.md` in your working directory. This contains the complete
product specification generated by the Planner.

### STEP 2: Create feature_list.json

Based on `app_spec.md`, create `feature_list.json` with detailed test cases.

**Format:**
```json
[
  {
    "id": 1,
    "category": "functional",
    "feature": "Feature name from spec",
    "description": "What this test verifies",
    "steps": [
      "Step 1: Navigate to page",
      "Step 2: Perform action",
      "Step 3: Verify result"
    ],
    "passes": false
  }
]
```

**Requirements:**
- Both "functional" and "style" categories
- Mix of narrow (2-5 steps) and comprehensive (10+ steps) tests
- Order by priority: foundational features first
- ALL start with `"passes": false`
- Cover every feature from the spec

**CRITICAL:** Features can ONLY be marked as passing later.
Never remove, edit descriptions, or modify steps.

### STEP 3: Create init.sh

Setup script for the development environment:
1. Install dependencies (npm, pip)
2. Start dev servers
3. Print access URLs

### STEP 4: Initialize Git

```bash
git init
git add .
git commit -m "Initial setup: feature_list, init.sh, project structure"
```

### STEP 5: Create Project Structure

Set up frontend (React+Vite) and backend (FastAPI) scaffold.

### STEP 6: Begin Implementation

Start implementing the highest-priority features. Work on ONE at a time.
Test thoroughly before marking `"passes": true`.

### Before Session Ends

1. Commit all work
2. Create/update `claude-progress.txt`
3. Ensure feature_list.json is saved
4. Leave environment clean
